# Webgenixx AI Caller Agent 🚀
### Outbound AI Cold-Calling CRM & Sales Automation System

Webgenixx AI Caller Agent is a production-ready system that automates your startup's outbound sales calling campaign. It allows you to upload lead lists from CSV spreadsheets, dynamically generate highly personalized sales pitches using Groq (Llama-3), place automated phone calls using Twilio Voice APIs, interact with customers using bilingual text-to-speech, store secure records inside MongoDB, and track real-time DTMF keypad responses inside an executive CRM dashboard.

---

## 🛠️ System Architecture

*   **Database Engine**: MongoDB (Mongoose ODM layer) with a fully automatic local JSON file cache fallback for 100% resilient offline testing!
*   **Frontend Client**: Flutter Mobile Agent App (Clean Architecture, Riverpod, Local Hive Database Cache, fl_chart Analytics).
*   **Backend Server**: Node.js + Express.js Server running on port `5005` (JWT Authentication, BCrypt encryptions, Groq AI Synthesizers, Twilio Voice Gateway).
*   **AIs**: Groq API Llama-3 Model.
*   **Call Router**: Twilio Voice API.

## ⚡ Quick Start

### 1. Boot Node.js Express Server
Open a terminal in the `backend/` directory:
```bash
# Enter directory
cd backend

# Install dependencies
npm install

# Run backend server in hot-reload development mode
npm run dev
```
The server will initialize on port `5005` (`http://localhost:5005`) and connect to MongoDB. If your local MongoDB daemon is not running, it will automatically launch in resilient local file database mode so the system never crashes!

It pre-seeds a default admin:
*   **Email**: `founder@webgenixx.com`
*   **Password**: `webgenixx123`

---

### 2. Sandbox Simulation Mode (Testing Out-of-the-Box)
If you do not have Twilio credits or Groq credentials yet, don't worry! 
By default, the system boots up in **Sandbox Simulator Mode**.
*   The server will simulate connection sequences, play synthetic pitch scripts, and complete calls after 8 seconds.
*   You will see **Sandbox Outcome Buttons** on the Calling Engine Cockpit during calling states, allowing you to manually click to simulate client key presses (Interested, Callback Later, Rejected).
*   This instantly updates your MongoDB database, charts, and metrics!

---

### 3. Exposing Call Webhooks (For Live Calls)
To connect real phone calls to Twilio, you need to expose your local backend server port `5005` to the public internet using `ngrok` so Twilio can send webhook callbacks:
```bash
# Launch ngrok forwarder
ngrok http 5005
```
1.  Copy the generated public HTTPS forwarding URL (e.g. `https://xxxx-xx-xx-xx.ngrok-free.app`).
2.  Open `backend/.env`.
3.  Add your Twilio credentials and paste the ngrok URL into the `BASE_URL` parameter (without trailing slash).
    ```env
    TWILIO_ACCOUNT_SID=ACyour_account_sid
    TWILIO_AUTH_TOKEN=your_auth_token
    TWILIO_PHONE_NUMBER=+1234567890
    GROQ_API_KEY=gsk_your_groq_key
    MONGODB_URI=mongodb://127.0.0.1:27017/webgenixx_ai_caller
    BASE_URL=https://xxxx-xx-xx-xx.ngrok-free.app
    ```
4.  Restart the Node server.

---

### 4. Build the Flutter Mobile Client App
Make sure your Android device is connected or an emulator is running:
```bash
# Download and link flutter packages
flutter pub get

# Run on your emulator/device
flutter run
```
*Note: The app is pre-configured to automatically connect to the local loopback address `http://10.0.2.2:5005` for Android emulators, making local sandbox testing completely plug-and-play!*

## 🏆 Key Features

*   **MongoDB Architecture**: Store secure login structures and campaign lead tables inside collection stores.
*   **Bilingual Scripts**: Synthetic Text-to-Speech pitches generated by Llama-3 in **English** or **Telugu** (incorporating respectful honorifics like *"garu"*).
*   **Duplicate Contact Filtering**: Cleans spreadsheets, normalizes number prefixes, filters invalid entries, and skips duplicate contacts automatically.
*   **Auto Campaign Loop**: The calling engine walks through pending leads sequentially, making calls, capturing keypad inputs, saving logs, and waiting a customizable pause timing before dialing the next lead!
*   **Executive Dashboard**: Displays Total Leads, Outbound Calls Completed, Pitches Interested, Conversion Ratios, and Daily Progress Bars.
*   **Weekly fl_chart Curves**: Interactive timeline charts mapping calls completed vs conversion outcomes over days of the week.
